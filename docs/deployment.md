# Deployment (Modrinth)

Fallow publishes to Modrinth with the [Mod Publish Plugin](https://github.com/modmuss50/mod-publish-plugin),
the same setup as the sibling Shulker Pocket project. One Modrinth version is uploaded per version
node (one today: 26.1.2). A `v*` git tag triggers the publish in CI; a local publish works too.

## One-time setup

1. **Create the Modrinth project** at <https://modrinth.com/dashboard/projects> -> New project.
   Use slug `fallow` (it matches `mod.modrinth_id` in `stonecutter.properties.toml`, the README
   badges, and the CI body-sync URL). Project type: Mod. Loader: Fabric. Environment: client
   optional, server required. Suggested categories: Game Mechanics, Decoration. Summary: the first
   line of the README. Upload `icon.png` as the icon.
2. **Generate a Modrinth PAT** at <https://modrinth.com/settings/pats> with the **Create versions**
   and **Write projects** scopes (the token previously in `.env` was expired).
3. **Add the token where it is needed:**
   - For CI publishing: GitHub -> repo Settings -> Secrets and variables -> Actions -> New
     repository secret named `MODRINTH_TOKEN`. (Or `gh secret set MODRINTH_TOKEN`.)
   - For local publishing: uncomment and paste it into `.env` (gitignored).

## Publish

- **Via CI (preferred):** push a version tag, e.g. `git tag v0.2.0 && git push origin v0.2.0`. The
  `build` workflow builds + tests, then the `publish` job uploads the jar to Modrinth and syncs the
  README as the project page. `workflow_dispatch` does the same on demand.
- **Locally:**
  ```sh
  set -a; source .env; set +a      # load MODRINTH_TOKEN
  ./gradlew chiseledPublishMods
  ```

Without a `MODRINTH_TOKEN` the publish **dry-runs**: it builds the jar and validates the config but
uploads nothing (`./gradlew publishMods` is a safe way to check the wiring).

## Files involved

- `build.gradle.kts` - the `publishMods { modrinth { ... } }` block (project id, game versions from
  `mod.mc_releases`, `requires("fabric-api")`, `optional("modmenu")`).
- `stonecutter.gradle.kts` - `chiseledPublishMods` fans the upload across every node.
- `stonecutter.properties.toml` - `mod.modrinth_id`, `mod.version`, per-node `mc_releases`.
- `.github/workflows/build.yml` - CI build + tag/dispatch publish + body sync.
- `.github/workflows/sync-body.yml` - re-sync the project page from the README without a new version.
- `icon.png` / `src/main/resources/assets/fallow/icon.png` - generated by `icon_art.py`.
